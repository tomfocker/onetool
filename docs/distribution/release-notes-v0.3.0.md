# OneTool v0.3.0

这次更新重点加入了日历应用、AI 日程助手和表格识别工具，同时修复了空间扫描、非默认安装目录启动和发布环境兼容性问题。

## 主要更新

- 新增 `日历应用`：山景玻璃拟态界面，支持日 / 周 / 月视图、拖拽空白创建日程、拖拽已有日程改时间、本地保存和同时间段日程并排展示。
- 新增 `AI 日历助手`：接入 OneTool 全局 LLM 配置，支持用自然语言创建日程，例如“明天下午3点安排设计评审”“后天上午有培训”。
- 新增 `表格识别`：支持从图片表格中识别内容并导出 Excel，包含本地运行时准备和状态提示。
- 优化 `空间扫描`：NTFS 极速扫描更稳，修复中文路径下极速扫描结果解析失败的问题；普通扫描默认只扫描前两级目录，并尽早展示大文件结果。
- 保留独立 `Docker 日历 WebUI`：适合后续独立部署或多端同步实验。

## 修复与优化

- 修复安装到非默认目录后可能启动无响应的问题。
- 修复主窗口加载异常时不显示窗口的问题，现在会自动兜底显示并记录日志。
- 修复日历周六、周日日程无法点选或修改的问题。
- 修复晚间日程超出默认时间范围后不易编辑 / 删除的问题。
- 修复 AI 创建日程后自动切换到日视图的问题。
- 移除日历助手中无实际功能的专注音乐 / Focus 建议卡片，避免误触噪音。
- 修复表格 OCR 安装脚本在非 UTF-8 Windows 控制台下可能失败的问题。
- CI / Release 环境统一到 Node 24，减少本地和云端构建差异。

## 升级提醒

- 当前 Windows 构建尚未代码签名，首次运行可能出现 SmartScreen 或“未知发布者”提示。
- 安装版支持应用内更新；便携版不支持应用内自动更新。
- 日历数据当前保存在本地，暂不支持云同步。
- AI 日历助手需要先配置可用的 OpenAI-compatible LLM 服务。
