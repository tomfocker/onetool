# v0.3.0 升级要点

`v0.3.0` 是一次偏向“日常办公工具扩展 + Windows 启动可靠性”的版本，在 `v0.2.0` 的 LLM 与发布基础上继续收口真实桌面场景。

## 重点升级

- 新增 `日历应用`
  - 山景玻璃拟态界面，适配 OneTool 默认工具箱宽度
  - 支持日 / 周 / 月视图、迷你月历、搜索、日历分类计数
  - 支持拖拽空白区域创建日程，拖拽已有日程平滑移动并在松手后落位
  - 支持自定义可见时间区间，晚间日程仍可点选、编辑和删除
  - 首次进入不再注入大量演示日程，本地日程保存在 OneTool 本地存储中
  - 同时间段多个日程会并排布局，避免互相覆盖
- 新增 `AI 日历助手`
  - 复用 OneTool 全局 OpenAI-compatible LLM 配置
  - 支持自然语言创建日程，例如“后天上午有培训”会识别为一整个上午
  - 创建日程后保持当前视图语境，不再强制切换到日视图
  - 移除无实际价值的专注音乐 / Focus 建议卡片，避免误触噪音和功能误导
- 新增 `表格识别`
  - 支持选择图片表格并本地识别
  - 支持导出 Excel
  - 内置运行时状态、安装准备、取消准备和结果打开链路
- 保留独立 `Docker 日历 WebUI` 版本
  - 适合作为单独 Web 应用部署
  - 包含 HTTP API、事件持久化和基础测试

## 可靠性修复

- 主窗口启动链新增兜底显示：如果渲染器没有触发 `ready-to-show`，2.5 秒后会显示窗口并记录日志
- 渲染器加载失败时会记录错误并显示主窗口，避免用户看到“点了没反应”
- 已用带空格和中文的非默认路径做 `win-unpacked` 冒烟验证，路径本身不会阻止应用启动

## 发布说明建议

- 当前仍为未签名 Windows 构建，首次运行可能出现 `SmartScreen` 或“未知发布者”提示
- 软件内自动更新仅适用于 Windows 安装版；便携版不支持应用内升级
- 建议发布前额外验证一次安装版自定义安装目录，例如 `D:\Apps\One Tool 测试`
- 如果用户反馈“安装到非默认位置后无响应”，优先让用户提供 `%APPDATA%\onetool\logs\main-YYYY-MM-DD.log`
