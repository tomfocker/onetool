# 空间清理第一版设计

**日期：** 2026-04-20

**状态：** 已确认，待评审

## 目标

新增一个独立的“空间清理”工具页，用于扫描单个磁盘或目录的空间占用情况，并通过目录树、Treemap 和大文件列表帮助用户快速定位可清理内容。

第一版目标是做出一个在当前 Electron 项目内可稳定运行、可视化清晰、可安全执行清理动作的磁盘分析工具，而不是在第一版里追求 `WizTree` 级别的极速扫描或底层 NTFS 专项优化。

## 用户意图与约束

基于本轮确认，用户希望：

- 有一个类似 `WizTree` / `WinDirStat` 的空间分析与清理工具。
- 能看到目录和文件的空间分布，而不只是一个列表。
- 能直接执行常用动作，例如打开位置、删除、重新扫描。
- 第一版先判断可行性并快速落地实用能力，不强求极限性能。

同时需要明确：

- 不直接复用 `WizTree` 源码或程序。
- 第一版允许“比 WizTree 慢”，但不能卡死 UI。
- 清理动作必须偏保守，避免误删造成不可恢复后果。

## 可行性结论

### 结论

第一版自研**可行**，难度属于**中等**，但要分层实现：

- “做出可用版本”可行
- “做出接近 WizTree 扫描速度”不适合作为第一版目标

### 原因

- 现有项目已经有成熟的 `main service -> IPC -> preload bridge -> renderer tool` 结构，适合增加主进程扫描类工具。
- React 渲染层已有足够 UI 能力，可直接实现目录树、列表和 SVG/Canvas 可视化。
- 主进程可以直接访问本地文件系统，适合做异步扫描和进度上报。

第一版真正的难点不在“能不能做”，而在：

- 大目录扫描时不能阻塞 Electron 主进程
- 结果数据量较大时不能把渲染层一次性压垮
- 清理动作需要安全边界

## 第一版范围

### 扫描对象

- 单个磁盘根目录，例如 `C:\`
- 单个任意目录，例如 `D:\Work`

第一版不支持：

- 多磁盘同时扫描
- 自动轮询监控
- 网络盘专项优化

### 展示内容

- 顶部总览
  - 扫描路径
  - 已扫描目录数
  - 已扫描文件数
  - 总大小
  - 最大文件
- 左侧目录树
  - 按大小排序的目录层级
  - 可展开/折叠
  - 支持钻取当前节点
- 主视图 Treemap
  - 当前目录层级的空间矩形图
  - 文件/目录按大小占比绘制
  - 点击节点可钻取或聚焦
- 大文件列表
  - 按大小降序
  - 显示文件名、路径、大小、扩展名

### 支持动作

- 选择扫描路径
- 开始扫描
- 取消扫描
- 重新扫描
- 在资源管理器中打开
- 复制路径
- 删除到回收站

### 第一版明确不做

- 重复文件检测
- 空目录批量清理
- 相似图片/视频识别
- 文件生命周期建议
- 权限提升删除
- `WizTree` 那种基于 NTFS 元数据的极限极速扫描

## 不采用外部源码的原因

### WizTree

`WizTree` 不是公开开源项目，其官方许可为 EULA 商业授权路线，而不是开放源码协议；其分发和再集成都需要额外授权，不适合直接嵌入当前项目。

### WinDirStat

`WinDirStat` 虽然公开源码，但当前公开仓库标注为 `GPL-2.0`。如果直接复用其源码进入本项目，会把当前项目带入 GPL 许可边界，风险和约束都明显增加。

因此第一版采用：

- 功能形态受 `WizTree` / `WinDirStat` 启发
- 数据结构、扫描逻辑、界面实现全部自研

## 设计原则

### 1. 扫描与渲染分离

扫描必须运行在主进程服务层，不能在渲染层递归文件系统。渲染层只消费结构化结果和进度事件。

### 2. 结果渐进显示

不能等整盘扫描结束才显示页面。第一版应支持：

- 先显示扫描中状态
- 逐步更新统计数字
- 在扫描完成后再切换到完整 Treemap 与完整排行

如果边扫边渲染 Treemap 会带来过高复杂度，则第一版允许：

- 扫描中只显示进度和阶段性摘要
- 扫描完成后一次性渲染 Treemap

### 3. 删除动作保守

第一版只允许“删除到回收站”，不做直接永久删除，不做批量强制删除。

### 4. 优先做“够快”

第一版目标不是和 `WizTree` 拼秒级整盘扫描，而是做到：

- 中小目录基本即时
- 常规开发机整盘扫描可接受
- 扫描过程中界面不假死

## 交互设计

### 工具定义

- 工具 ID：`space-cleanup`
- 工具名：`空间清理`
- 分类：`系统维护`

### 页面结构

#### 顶部控制区

- 路径输入框或只读显示框
- `选择目录`
- `扫描本盘` / `重新扫描`
- `取消扫描`
- 当前状态标签：
  - 空闲
  - 扫描中
  - 已完成
  - 已取消
  - 扫描失败

#### 顶部总览区

展示 4-6 个紧凑统计卡：

- 扫描大小
- 文件数
- 目录数
- 最大文件
- 扫描耗时

#### 主工作区

左右布局更合适：

- 左：目录树
- 中：Treemap
- 右：详情面板 / 大文件列表

### Treemap 交互

第一版采用最小可用交互：

- Hover 显示名称、路径、大小、占比
- 点击目录进入下一层
- 点击文件聚焦详情
- 顶部 breadcrumb 支持返回上一级

### 清理动作入口

文件或目录选中后显示：

- `打开位置`
- `复制路径`
- `删除到回收站`

删除前必须二次确认。

## 数据模型

### 扫描节点

建议主进程维护统一树节点：

- `id`
- `name`
- `path`
- `type` (`file` / `directory`)
- `sizeBytes`
- `childrenCount`
- `fileCount`
- `directoryCount`
- `extension`（文件节点）
- `children`（目录节点）

### 扫描会话

建议主进程维护一次扫描会话摘要：

- `sessionId`
- `rootPath`
- `status`
- `startedAt`
- `finishedAt`
- `scannedFiles`
- `scannedDirectories`
- `totalBytes`
- `largestFile`
- `tree`
- `largestFiles`

### 前端视图模型

前端不直接操作原始扫描过程，消费整理后的视图模型：

- 总览统计
- 当前目录 breadcrumb
- 当前目录子节点列表
- Treemap 可视化节点数组
- 大文件扁平列表

## 主进程设计

### 新增服务

建议新增：

- `SpaceCleanupService`

职责：

- 发起目录扫描
- 管理取消状态
- 统计树节点大小
- 生成总览摘要
- 生成大文件排行
- 对外提供打开位置、回收站删除等动作

### 扫描策略

第一版使用标准文件系统递归扫描：

- `fs.promises.readdir`
- `fs.promises.stat` 或等价方案

要求：

- 全程异步
- 按批次让出事件循环
- 支持取消 token

为了降低复杂度，第一版可以先不做 worker thread，但要在实现上保留未来迁移可能。

### 性能控制

第一版建议增加 3 个限制：

- 单次并发目录读取数限制
- 单次批处理后主动让出事件循环
- 大文件排行只保留前 N 条，例如前 500 条

这能明显降低主进程被长任务拖慢的风险。

### 回收站删除

删除动作建议通过 Electron / Windows 可用能力实现“移入回收站”，而不是直接 `fs.rm`。

如果仓库当前没有现成的回收站删除能力，第一版可以：

- 先用平台命令或轻量依赖实现
- 明确失败时给用户错误提示

不允许静默回退为永久删除。

- `space-cleanup-progress`

## 渲染层设计

### 新增工具页

建议新增：

- `src/renderer/src/tools/SpaceCleanupTool.tsx`

### 相关 hook

建议新增：

- `useSpaceCleanup.ts`

职责：

- 管理扫描状态
- 管理当前路径与当前节点
- 管理 Treemap hover / selection
- 消费主进程进度事件

### 可视化实现

第一版优先使用自绘 `SVG` Treemap，而不是先引入大型图表库。

原因：

- 当前项目里没有成熟图表依赖
- Treemap 交互目标有限
- 自绘 SVG 更容易控制样式、hover、点击和暗色/亮色统一

布局算法第一版可用“切片矩形”或“简化 squarified treemap”，不必一开始追求最优美布局。

## 错误处理

- 无权限目录：
  - 跳过
  - 记录统计
  - 在扫描完成摘要里提示“部分目录未读取”
- 路径不存在：
  - 直接失败并提示
- 扫描被取消：
  - 保留已扫描统计
  - 不渲染不完整 Treemap，或明确标记“结果不完整”
- 删除失败：
  - 保持当前结果不变
  - 显示明确错误

## 测试策略

### 主进程测试

至少覆盖：

- 扫描树大小累加正确
- 空目录、单文件目录、多层目录结果正确
- 大文件排行正确排序
- 取消扫描能中断后续遍历
- 无权限/异常路径被正确跳过或报错
- 删除动作只走回收站删除，不走永久删除

### 渲染层测试

至少覆盖：

- 总览统计映射正确
- 当前节点切换能驱动 Treemap 与目录树同步
- 扫描中、完成、取消、失败状态切换正确
- 空结果、部分结果、错误结果展示正确
- Treemap 选中后动作按钮显示正确

## 验证标准

第一版完成标准：

- 可选择单个目录或盘符进行扫描
- 扫描过程中应用不假死
- 扫描完成后能展示目录树、Treemap、大文件列表
- 支持打开位置、复制路径、删除到回收站
- 取消扫描可生效
- 现有测试与构建通过
