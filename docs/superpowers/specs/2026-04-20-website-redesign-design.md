# OneTool 官网改版设计

**日期：** 2026-04-20

**状态：** 已确认，待评审

## 目标

对 `website/` 目录下的官网进行一次面向发布页的视觉与信息架构改版，让首页在保持高级感和记忆点的同时，明显提升可理解性、素材完成度与滚动体验稳定性。

这次改版不是继续放大“概念页”表达，而是把官网收束成一个更像正式产品发布页的入口：普通 Windows 用户可以在数秒内看懂 OneTool 是什么、能做什么，同时仍然能感受到明确的视觉个性。

## 用户意图与约束

基于本轮确认，用户希望：

- 官网整体更高级、更有记忆点。
- 不再让首页文案显得抽象、绕、难懂。
- 首页主视觉以真实软件界面截图为主，而不是纯抽象图形。
- 首页首先服务普通 Windows 用户，而不是只服务进阶用户。
- 首屏先强调“一个应用覆盖日常高频工具”的价值。
- 保留用户滚动时“晶体 / 功能层逐步破碎展开”的 3D 视觉体验。
- 去掉当前难以解释的黑色视频，并降低滚动卡顿。

## 核心问题

当前官网存在四个核心问题：

1. 文案概念感过重，用户不能快速理解产品价值。
2. 视觉素材数量和质量不足，导致页面主要依赖抽象表达撑气质。
3. 首屏使用黑色视频作为背景，但它没有承担清晰的信息职责，反而分散注意力。
4. 首屏长段 sticky 滚动拆解动画过重，影响性能和流畅度。

## 改版方向

本次改版采用“软件界面发布页”方向：

- 页面气质参考硬件发布页，但服务对象仍然是软件产品。
- 首页不再使用“控制晶体”作为主标题或核心命名。
- 高级感主要来自版式、材质、构图、留白和动效节奏，而不是抽象概念词。
- 主视觉用一组经过统一处理的软件界面卡片建立记忆点。

## 非目标

- 不把官网改成纯软件下载站模板。
- 不引入真实 3D 引擎、粒子系统或物理碰撞式破碎效果。
- 不为了展示系统深能力而让首页重新变得技术导向。
- 不要求一次性补齐所有工具的精修宣传图。
- 不继续使用无明确作用的视频背景。

## 设计原则

### 1. 先让用户看懂，再让用户感到“高级”

首页必须先完成“这是什么产品”的信息传达，再通过视觉风格建立高级感。高级感不能建立在理解成本之上。

### 2. 主视觉必须来自真实产品，而不是纯概念图

官网主视觉以真实软件界面截图为基底，再通过统一的卡片容器、光感、景深、层叠关系做发布页化处理。这样既能保证真实感，也能避免空洞。

### 3. 保留滚动记忆点，但把效果控制在可维护范围内

首屏保留“整体组合体随着滚动逐层拆开”的体验，但实现方式应为轻量假 3D 视觉错觉，而不是重型 3D 物理演出。

### 4. 首屏讲日常高频价值，深系统能力后置

首屏和前两屏优先讲普通用户能立即共鸣的功能，例如截图、录屏、重命名、剪贴板、二维码等。代理、WSL 等进阶能力保留，但应后置到中后段。

### 5. 少量高质量素材胜过大量低质量占位

比起堆更多抽象块或临时占位，更重要的是把少数真实界面素材打磨成一套风格统一的官网资产。

## 信息架构

首页改版后建议收束为五段：

### 1. Hero 首屏

作用：

- 第一时间解释 OneTool 是什么。
- 建立“高级桌面产品”的第一印象。
- 提供下载或前往 GitHub 的直接入口。

内容：

- 简洁主标题，例如“一个应用，收齐 Windows 日常高频工具。”
- 一句副标题，补充产品范围和价值。
- 主行动按钮：下载 / GitHub。
- 右侧或中部主视觉：由真实软件界面截图组成的组合体。

### 2. 核心价值条

作用：

- 在首屏之后快速解释为什么值得使用。

内容建议：

- 常用功能更集中
- 界面更清爽
- 本地运行更安心

每条价值只保留一句短解释，不展开成长段文案。

### 3. 日常高频场景

作用：

- 按普通用户的语言讲功能，不用抽象分类名词。

建议分为三类：

- 截图与录屏
- 文件整理与批量重命名
- 剪贴板、二维码等效率小工具

每张场景卡包含：

- 一张界面图
- 一个清晰标题
- 一句简短价值说明
- 2 至 4 个代表性功能点

### 4. 工具总览

作用：

- 告诉用户 OneTool 不只是几个工具，而是一整个工具箱。

建议按用户语言分组：

- 文件处理
- 图像与屏幕
- 效率小工具
- 系统辅助

这里以矩阵或标签分组呈现，强调覆盖面，但不进入复杂说明。

### 5. 下载与信任收口

作用：

- 完成下载转化。
- 用简短事实建立信任。

内容建议：

- 支持 Windows
- 本地运行
- GitHub 开源
- 下载按钮与源码入口

## 首屏视觉设计

### 文案原则

首屏文案应满足以下限制：

- 主标题 1 句，优先人话，不用概念命名。
- 副标题最多 2 句。
- 避免“重铸”“晶体”“核心”“深水区”等抽象隐喻主导信息。
- 用词更贴近日常使用场景，而不是品牌 campaign 语言。

### 主视觉结构

首屏主视觉应由以下部分组成：

- 一个最大的“主界面核心层”，代表 OneTool 主体。
- 3 至 5 个围绕主界面的功能卡片层。
- 少量柔光、玻璃边缘、高光和薄雾氛围层。

卡片素材优先选择最容易理解的工具：

- 截图 / 录屏
- 批量重命名
- 剪贴板管理
- 二维码
- 可选补充：取色器或图片处理

这些素材不是简单截图堆叠，而应统一成官网用卡片体系：

- 一致的圆角
- 一致的描边和投影
- 一致的背景明度
- 一致的透视关系和层叠节奏

## 首屏滚动动效

### 动效目标

保留“用户滚动时，整体主视觉逐层破碎展开”的记忆点，但把实现控制在高性能、低复杂度范围内。

### 实现策略

采用“轻量假 3D 分层展开”，而不是物理碎裂：

- 使用若干分层卡片与装饰层组成完整组合体。
- 随滚动进度对每层应用 `translate3d`、`rotateX`、`rotateY`、`scale`、透明度和轻微模糊变化。
- 主界面核心层位移最小，作为视觉锚点。
- 外围功能卡片按层级向外散开，营造破碎展开感。

### 分层建议

首屏建议至少拆为五层：

1. 背景氛围层
2. 主界面核心层
3. 日常工具层
4. 补充功能层
5. 文案与按钮层

### 滚动节奏

- `0% - 25%`
  用户先看到一个完整的产品组合体。
- `25% - 65%`
  功能卡片按层级展开，形成明显的 3D 破碎感。
- `65% - 100%`
  主视觉逐渐退场，把注意力让给后续内容区。

### 性能边界

为避免重复出现当前卡顿问题，动效实现应遵守：

- 不使用视频背景。
- 不引入 WebGL 或 3D 引擎。
- 优先使用 transform 和 opacity，不使用高成本布局动画。
- 控制同时运动的层数和单层阴影复杂度。
- 为 `prefers-reduced-motion` 提供降级版本。

## 素材策略

### 必需资产

第一版至少需要准备以下素材：

- 3 至 5 张真实界面截图，用于首屏与场景卡片。
- 1 张统一气质的氛围背景图或光感底图。
- 下载区或工具总览需要的少量图标与标识。

### AI 生图使用策略

AI 生成图片只承担以下职责：

- 生成首屏或局部版块需要的氛围背景。
- 为真实截图提供统一视觉环境。
- 补足页面中缺少的发布页感材质。

AI 图不承担以下职责：

- 伪造软件界面主体。
- 替代真实工具截图。
- 让页面重新走向难理解的抽象概念图。

## 内容策略

### 首页优先讲的内容

- 一个应用覆盖多种日常高频工具。
- 工具集中，减少到处找入口。
- 视觉更统一，使用更顺手。
- 本地运行，下载即可用。

### 中后段再讲的内容

- 代理管理
- WSL 管理
- 更偏系统层的辅助能力

这些能力可以作为“除了日常工具，也能处理更进阶桌面环境能力”的补充段落，而不是首页主线。

## 组件与实现边界

改版范围聚焦于 `website/` 静态站：

- `index.html`
- `style.css`
- `script.js`
- 官网引用的图片和视频等资产

不在这次设计中引入新的前端框架、构建系统或大型运行时依赖。

### 建议的实现重点

- 重写 Hero 结构与文案。
- 替换视频背景为静态 / 图像型主视觉。
- 重构首屏动画逻辑，从长段 sticky 重演出改为更克制的分层展开。
- 重新组织后续版块的标题、顺序和卡片样式。
- 对官网使用的截图资产进行统一裁切和风格化处理。

## 验证标准

完成实现后，改版应至少满足以下结果：

- 用户首次进入页面能在 5 秒内看懂 OneTool 是桌面工具箱。
- 首屏高级感主要来自截图主视觉与版式，而不是抽象文案。
- 页面不再出现意义不明的视频背景。
- 滚动展开效果仍然存在，但性能明显优于当前版本。
- 首页前两屏以普通用户可理解的功能为主。
- 页面在桌面端与移动端都能稳定阅读与操作。

## 风险与取舍

### 1. 素材质量不足会削弱整个方案

本方案高度依赖真实截图主视觉。如果截图质量不统一，页面会失去发布页质感。因此资产整理与统一处理是改版成功的关键工作之一。

### 2. 动效层级过多会重新带来性能问题

虽然保留分层展开，但必须严格限制层数和视觉复杂度，否则容易回到当前卡顿状态。

### 3. 过度追求“高级感”会再次牺牲信息效率

如果后续实现中重新加入过多概念词、装饰块或难解释的视觉元素，会让页面再次偏离这次确认的方向。

## 结论

本次官网改版应把 OneTool 从“抽象概念页”调整为“以真实界面为主的高级软件发布页”。

页面需要继续保留滚动时的 3D 分层展开记忆点，但表达方式必须从重演出改成高性能、低理解成本的视觉错觉。首页内容则应回到最直接的用户价值：一个应用，覆盖 Windows 日常高频工具。
